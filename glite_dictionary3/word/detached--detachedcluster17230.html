detached

adjective
in version control: not on a named branch
Pronunciation: / dɪˈtætʃt / (AmE), / dɪˈtætʃt / (BrE)
(in computing, especially version-control systems) describing a repository state where the current checkout (for example, HEAD in Git) is not on a named branch but points directly to a commit.

Examples

detached HEAD state
In Git, a detached HEAD means you're not on any named branch and commits may be lost.
She accidentally left the repository in a detached state after checking out an old commit.
To fix a detached HEAD, you can create a new branch or switch back to an existing branch.

Related Terms

  • antonym – attached: in version control: connected to a named branch (opposite of detached state)
  • related – branch: a named line of development in a version-control system
