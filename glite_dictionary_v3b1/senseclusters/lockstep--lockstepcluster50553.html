lockstep

noun
synchronized execution for fault tolerance
8.2
Pronunciation: /ˈlɑːk.step/ (AmE), /ˈlɒk.step/ (BrE)
(computing, engineering) a fault-tolerant operating mode in which two or more processors or execution units run identical instructions at the same time so that errors can be detected or masked by comparing results.

Examples

Lockstep processing
The safety-critical system runs in lockstep to detect and isolate hardware faults quickly.
Engineers configured the flight controller so two processors would execute in lockstep during tests.
When one core produced a different result, the monitoring software flagged the lockstep pair for inspection.

Related Terms

  • synonym: fault tolerance - the ability of a system to continue operating properly in the event of the failure of some of its components
  • synonym: redundancy - the inclusion of extra components that perform the same function to increase system reliability
