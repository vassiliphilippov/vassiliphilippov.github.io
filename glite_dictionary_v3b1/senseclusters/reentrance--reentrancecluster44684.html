reentrance

noun
software routine safety
9.5
Pronunciation: /riː.ˈen.trəns/ /; / /rɪ.ˈen.trəns/ (AmE), /riː.ˈen.trəns/ /; / /rɪ.ˈen.trəns/ (BrE)
the ability of a software routine to be safely called again before previous executions have finished, especially in concurrent programming.

Examples

Software reentrance
The function's reentrance is crucial for handling multiple user requests at the same time.
Without reentrance, the program could crash when two threads call the same routine simultaneously.
Developers tested the reentrance of the code to ensure it worked correctly in a multithreaded environment.

Related Terms

  • antonym: non-reentrant - not safe to call again before previous executions have finished
  • synonym: thread-safe - able to function correctly when accessed by multiple threads at the same time
