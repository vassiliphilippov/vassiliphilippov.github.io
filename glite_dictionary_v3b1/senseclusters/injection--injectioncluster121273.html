injection

noun
software design pattern (dependency injection)
5.7 0.95/m
Pronunciation: / \u026an\u02c8d\u0292\u025bk\u0283\u0259n / (AmE), / \u026an\u02c8d\u0292\u025bk\u0283\u0259n / (BrE)
(in software engineering) a design pattern in which an object's required components or services (its dependencies) are supplied from the outside rather than created by the object itself.

Examples

Dependency injection pattern
The framework uses injection to supply services to components, promoting loose coupling.
Unit tests were easier because the developers relied on injection to provide mock objects.
After refactoring, the codebase adopted injection, allowing implementations to be swapped at runtime.

Related Terms

  • synonym: dependency injection - the software pattern of providing an object's dependencies from the outside
  • synonym: inversion of control - a related design principle where control of component creation is inverted from the object to an external system
