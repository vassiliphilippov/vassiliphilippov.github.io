Shellshock

noun
2014 Bash security vulnerability
Pronunciation: / ʃel ʃɑk / (AmE), / ʃel ʃɒk / (BrE)
A security vulnerability in the Bash shell (disclosed in 2014) that allowed attackers to execute arbitrary commands through specially crafted environment variables.

Examples

Shellshock vulnerability
System administrators rushed to patch the Shellshock vulnerability across thousands of servers.
The security team explained how Shellshock allowed attackers to run commands through crafted environment variables.
After the 2014 disclosure, many vendors released patches to fix Shellshock on affected systems.

Related Terms

  • synonym: Bash vulnerability - a security flaw in the Bourne Again Shell (Bash) allowing unintended code execution
