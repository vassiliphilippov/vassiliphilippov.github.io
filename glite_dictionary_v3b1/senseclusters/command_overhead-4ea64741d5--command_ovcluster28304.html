command overhead

noun
processing time required before command execution (computing)
Pronunciation: / kəˈmænd ˈoʊvɚˌhɛd / (AmE), / kəˈmɑːnd ˈəʊvəhɛd / (BrE)
(computing) the amount of processing time or resources required by a device or system before it begins to execute a given command, often considered as a measure of inefficiency or processing cost.

Examples

Command overhead time
Engineers analyzed the command overhead to optimize the performance of the embedded system.
Reducing command overhead can greatly improve the overall efficiency of a device.
The command overhead of the operating system increased response times for simple tasks.
