suspension point

noun
point where a coroutine suspends
9.1
Pronunciation: /sə.ˈspen.ʃən pɔɪnt/ (AmE), /sə.ˈspen.ʃən pɔɪnt/ (BrE)
(in concurrent programming) a point in a program where a coroutine or suspendable function suspends execution, yielding control so it can be resumed later.

Examples

Coroutine suspension point
In Kotlin, a suspension point occurs when calling a suspend function.
The debugger stopped at a suspension point while the coroutine waited for network data.
Designing clear suspension points helps avoid bugs in concurrent coroutine code.

Related Terms

  • antonym: resume - to continue execution of a previously suspended coroutine or function
  • synonym: yield point - a point where execution yields control, often used interchangeably with suspension point in coroutine contexts
