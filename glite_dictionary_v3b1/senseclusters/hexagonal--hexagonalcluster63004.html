hexagonal

adjective
relating to hexagonal software architecture
7.4
Pronunciation: /hek.ˈsæ.gə.nəl/ (AmE), /hek.ˈsæ.gə.nəl/ (BrE)
(in software engineering) relating to or using hexagonal architecture, a design pattern that isolates core application logic from external systems by using ports and adapters.

Examples

Hexagonal architecture
The team adopted hexagonal architecture to keep core logic independent from external interfaces.
Using a hexagonal design allowed their microservices to be tested without depending on databases or UI.
Many engineers prefer hexagonal patterns because they separate application rules from delivery mechanisms cleanly.

Related Terms

  • synonym: hexagonal architecture - a software design pattern that decouples core logic from external interfaces using ports and adapters
  • synonym: ports and adapters - an approach to organising software so the core logic communicates with the outside world through defined ports and adapter implementations
